# CRM для рекрутеров (v0.0.1)

Локальное десктоп‑приложение на Python/Tkinter для хранения рекрутеров с базой SQLite, логами и ссылками на Telegram/резюме.

## Что умеет
- Добавление рекрутера: компания, ФИО, ник в TG, телефон, должность, почта, комментарии, PDF‑резюме.
- Просмотр и фильтр по компании, двойной клик по никнейму открывает Telegram.
- Кнопки: открыть TG, открыть резюме, удалить запись.
- Данные сохраняются в `crm.db` (SQLite), логи действий — в `logs/app-*.log`.
- При открытии Telegram параллельно открывается PDF резюме, чтобы его можно было сразу прикрепить вручную.

## Запуск
### Самый быстрый способ (Windows, один клик)
- Дважды кликнуть `run.bat` в папке `petProjects/CRM`.  
  Скрипт сам создаст venv (если нет) и запустит приложение.

### Альтернативно (macOS/Linux)
```bash
cd petProjects/CRM
./run.sh             # при необходимости: chmod +x run.sh
```

> Требуется установленный Python 3.10+. Внешних зависимостей нет.

## Где лежат данные
- База: `crm.db` (в корне проекта). Это и есть резервная копия данных.
- Логи: `logs/app.log` (с ротируемыми файлами по датам).
- Резюме: хранятся как путь к выбранному PDF. Файлы остаются там, где вы их выбрали.

## Быстрые подсказки по UI
- Заполните поля → «Добавить рекрутера».
- «Показать рекрутеров» — обновить таблицу (с учётом фильтра по компании).
- Двойной клик по колонке TG или кнопка «Открыть TG» — открывает Telegram (`tg://…` или `https://t.me/…`) и сразу открывает PDF резюме.
  > Полностью автоматическое прикрепление файла в Telegram без токена бота недоступно, поэтому файл открывается локально, чтобы быстро его выбрать.
- «Открыть резюме» — открывает прикреплённый PDF.
- «Удалить выбранного» — удаляет запись из базы.

## Отладка
- Если что-то пошло не так, посмотрите свежий лог в `logs/` — поиск по последней дате.
