# CRM для рекрутеров (v0.0.1)

Локальное десктоп‑приложение на Python/Tkinter для хранения рекрутеров с базой SQLite, логами и ссылками на Telegram.

## Что умеет
- Добавление рекрутера: компания, ФИО, ник в TG, телефон, должность, почта, комментарии, статус, даты (последний контакт, следующий шаг).
- Просмотр и фильтр по компании и статусу; сортировка по «следующему шагу».
- Двойной клик по никнейму открывает Telegram; кнопка «Открыть TG» тоже.
- Кнопка удаления записи.
- Данные сохраняются в `crm.db` (SQLite), логи действий — в `logs/app-*.log`.

## Запуск
### Самый быстрый способ (Windows, один клик)
- Дважды кликнуть `run.bat` в папке `petProjects/CRM`.  
  Скрипт сам создаст venv (если нет) и запустит приложение.

### Альтернативно (macOS/Linux)
```bash
cd petProjects/CRM
./run.sh             # при необходимости: chmod +x run.sh
```

> Требуется установленный Python 3.10+. Внешних зависимостей нет.

## Где лежат данные
- База: `crm.db` (в корне проекта). Это и есть резервная копия данных.
- Логи: `logs/app.log` (с ротируемыми файлами по датам).
- Резюме: хранятся как путь к выбранному PDF. Файлы остаются там, где вы их выбрали.

## Быстрые подсказки по UI
- Заполните поля → «Добавить рекрутера». Статус по умолчанию — «первичный контакт».
- Поля дат вводятся вручную в формате `YYYY-MM-DD` (например, `2025-12-05`).
- «Показать рекрутеров» — обновить таблицу с фильтрами (компания, статус).
- Двойной клик по колонке TG или кнопка «Открыть TG» — открывает Telegram (`tg://…` или `https://t.me/…`).
- «Удалить выбранного» — удаляет запись из базы.

## Отладка
- Если что-то пошло не так, посмотрите свежий лог в `logs/` — поиск по последней дате.
